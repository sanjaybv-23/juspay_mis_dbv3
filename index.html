Where Does the Variance Reasoning Come From?

Every insight on this dashboard is generated by reading your master Excel file and applying a set of rules. Nothing comes from external AI or web data — all numbers and conclusions trace back to specific cells in your workbook.

1. Direct numerical comparison

For each line item (Revenue, Hypercheckout, Cloud Cost, etc.) the dashboard pulls three values from the MIS_FY27 sheet: Actual FY27 (cols E-P), AOP FY27 (cols S-AD), Actual FY26 (cols AH-AS). Variance = LHS − RHS based on your "Compare" selection.

2. Direction logic (favourable vs adverse)

Revenue lines: higher actual vs benchmark = favourable (green). Cost lines (anything in Direct Cost or Operating Cost section): lower actual vs benchmark = favourable (green). The dashboard knows which lines are costs based on which section header they sit under in the source sheet.

3. Outlier detection

For each line, the dashboard computes the median of monthly values. Any month with value > 5× the median is flagged as a likely one-off (e.g., the ₹84 Cr International Bank Business spike in Sep — that's why it appears in commentary as "likely a one-time EU contract recognition"). This protects trend analysis from being distorted by exception items.

4. Domain heuristics

Line-item-specific reasoning hints are baked in. Examples:

  • PA → "PA gross margin is structurally near-zero — growth here is volume-driven."
  • Indirect Cloud Infrastructure → "Cloud overruns usually = unmonitored auto-scaling or new workloads launched without capacity planning."
  • Gaming industry → "Gaming sector facing regulatory headwinds (28% GST on full face value from Oct 2023)."
  • Hypercheckout → "Check merchant churn (especially top-10) and conversion rate trends."

These are pattern-matched on line-item label — no live web lookups, no external API calls.

5. What's NOT used (yet)

  • Transaction count and GMV from UPI_Data, Merchant Business, Breeze sheets — these would enable richer reasoning like "revenue down but volume up = take-rate compression". Coming next iteration.
  • Top-10 merchant data from Top10_Jul, Top_10Mar26 — would enable "merchant X drove Y% of decline" analysis.
  • Forecast (FC) sheet vs AOP — would enable "FC is now Z% below AOP, revise plan?" alerts.

When you don't agree with the reasoning

Click Edit on any insight card. Your override saves locally. Use Export Comments to share via JSON (or, in live mode with shared comments, automatically with your team).

Editing Data & Commentary

For underlying numbers: Always edit your master Excel / Google Sheet. The dashboard is a viewer, not the source of truth — keeping data in one place avoids divergence and preserves audit trail.

For variance commentary: Hover any insight card and click Edit. Type your own reasoning. It saves to your browser. Use Export Comments to share with team via JSON, and Import Comments to load a teammate's overrides.

For new lines / sections: Add them in the master Excel. The dashboard auto-detects most structural changes when data refreshes.

Live Mode Setup
  1. Upload Testing_Revenue_Data_SM.xlsx to Google Drive → Open with Sheets → Save as Google Sheets.
  2. File → Share → Publish to web → MIS_FY27 tab → CSV format → copy URL.
  3. Edit this HTML, find SHEET_URL, paste the URL, save, re-deploy.

Variance Reasoning Methodology

Auto-commentary uses four signals: magnitude (absolute deviation), direction (favourable/adverse), outliers (months >2× the median, flagged as one-offs), and trend (consecutive same-direction months).

The Focus Areas view answers two specific questions: "which segments/merchants are underperforming and why" and "where are we overspending vs plan". Use this for MIS reviews — it's designed to be the at-a-glance answer.
